Embedded Computer Systems (ECS)

Fall 2009

 
 

The line between general purpose programming and low-level specialized programming is blurring as systems that used to be specialized and low-level (like real-time systems) become larger and more complex. Managed, type-safe, garbage-collected languages like Java and C# are increasingly attractive candidates for writing large complex systems. So, it is natural to think of applying them to real-time systems. But, are they up to the job? The goal of this course is to introduce student to the challenges of real-time programming and introduce Java as a general-purpose language capable of solving hard real-time problems.


Participants interested in real-time will gain a tool that will be immediately useful in their own research. Participants who are merely curious about embedded programming will get a survey of the ﬁeld.


The course will cover the following topics:


  1. Concurrent programming

  2. Real-time programming

  3. Worst case execution time analysis

  4. Schedulability Analysis

  5. Real-time garbage collection

  6. Safety critical system verification and validation

  7. Model-driven architectures


Participants will gain practical experience with the following technologies:


  1. RTEMS real-time operating system

  2. LEON radiation-hardened processors

  3. Real-time Java programming

  4. Static program analysis for worst case execution with Rapitime


Case studies will be selected in the fields of avionics and space applications.


The workload will be modular, students will be allowed to choose between a theory-only and hands-on set of projects.
